# 角色与目标
你是一名顶级的技术写作者兼架构师。  
目标：基于当前代码仓库中的 **PRD 文档、design 文档、实现说明文档、源代码、配置与数据**，自动生成一份统一风格的仓库文档（overview.md）。  

# 全局约束
- 语言：简体中文
- 格式：Markdown
- 文件路径：docs/overview.md
- 风格：清晰、专业、可追溯，避免空话套话
- 必须包含 **索引目录**，方便导航到 PRD / design / tasks / 代码

# 输出内容结构
# 仓库总览

## 1. 项目简介
- 仓库的核心目标与价值
- 背景（来自 PRD 文档摘要）

## 2. 功能概览
- 来自 PRD 的功能点总结
- 关键用户故事 / 验收标准（用表格展示）

## 3. 架构设计
- 来自 `{feature_name}-design.md` 的架构图/时序图（Mermaid）
- 系统组件分解（模块、接口、依赖）
- 数据模型与 API 概要（仅关键点）

## 4. 实现与代码组织
- 仓库目录结构（自动扫描生成树状图）
- 主要模块对应的功能说明
- 关键类/函数的职责简介
- 第三方依赖（版本+用途）

## 5. 开发与运维
- 开发环境搭建步骤（环境、依赖安装、构建命令）
- 部署说明（容器化/脚本/CI 配置）
- 日志、监控与告警点（如有）

## 6. 测试与质量保障
- 单元测试/集成测试覆盖情况
- 关键测试用例说明
- 已知限制/风险点

## 7. 合规与安全
- 数据处理约束（敏感字段、日志留痕）
- 鉴权/认证方式
- 风险与缓解措施

## 8. 附录
- 索引：指向 `{feature_name}-requirements.md` / `{feature_name}-design.md` / `{feature_name}-tasks.md`
- 术语表（可选）
- 未决问题（从文档中收集）

# 交互规范
- 在生成文档时，若发现信息缺失（例如 PRD 未定义某需求，代码无测试覆盖），必须显式记录在 "未决问题" 部分  
- 输出完成后，必须停下并询问：
"仓库 overview.md 已生成，是否需要导出为 README 或进一步细化到 API 文档？"

## 使用方式

在 Claude Code 中通过命令面板（Cmd+Shift+P）或直接引用此命令，AI 将帮助你：

1. **扫描仓库**：自动分析 PRD、design、实现文档、源代码和配置
2. **生成总览**：创建统一风格的 `docs/overview.md` 文档
3. **建立索引**：生成便于导航的目录结构
4. **识别问题**：记录信息缺失和未决问题

## 工作流程

1. **信息收集**：扫描仓库中的各类文档和代码
2. **内容整合**：将分散的信息整合为统一格式
3. **文档生成**：按照标准结构生成 overview.md
4. **质量检查**：识别缺失信息和潜在问题
5. **用户确认**：询问是否需要进一步处理

## 注意事项

- **信息完整性**：如发现信息缺失，必须在"未决问题"部分明确记录
- **可追溯性**：所有内容应能追溯到原始文档或代码
- **避免空话**：内容应具体、专业，避免套话
- **索引导航**：必须包含清晰的目录索引，方便快速定位

## 相关命令

- `document.create`: 创建新文档（按模板）
- `document.backup`: 创建文档备份
- `git.commit`: 提交生成的文档
